## 3. Review findings

### 3.1 Theory and checklist fidelity — passed

The brief kept the critical distinctions open until their declared decision
points and the final records preserved them:

```text
niche medium state
!= niche-conditioning relation
!= niche formation
```

C02 was described as a bounded writer-medium-reader relation under explicit
constructed support. It was not relabeled as native niche formation, agency,
coordination, organism, motif, or regime. The result report retained the raw
`[1,0,1,0]` distribution, threshold relation, selectivity interaction, support
status, and blocked claims rather than projecting only the maximal normalized
margin.

Checklist growth improved discrimination. The only post-execution successor,
C02, followed the universal C01 restoration failure and changed one RCAE
integrity predicate under `P2-I1-DEC-027`. It preserved the scientific matrix,
analysis identity, controls, seeds, attempts, and claim ceiling. No outcome was
used to tune a threshold, add a rescue cell, or change the desired terminal
class.

### 3.2 Calibration provenance — sufficient

The matched-null chain retained exact generator and input provenance,
candidate/runtime exclusion assertions, five disjoint calibration seeds, all
five possible formation fractions, zero paired null margins, and the numeric
resolution floor `delta=1e-12`. Calibration and live analysis used the same
frozen policy path.

This was sufficient to establish measurement resolution without importing
candidate behavior. No calibration meaning or core schema needs to reopen.

### 3.3 Registration evidence — sufficient, bespoke

Registration did not rely on `validate-phase1`. The retained bundle joined the
metric and calibration identities, realization profile, local PyGRC binding,
baseline registry, inherited-source verification, control plans, exact source
revision, reconstruction commands, and artifact manifest. The manifest is
fully resolved and all fifteen indexed artifacts passed their retained
identity checks.

The implementation was extensive, but most complexity came from the first
live PyGRC binding, exact baseline and source identity, and lane-specific
causal controls. A new `lane_registration` schema would not remove those
semantic validators. The existing artifact manifest, realization profile,
runtime receipt, calibration records, and lane-local freeze represented and
verified the boundary without ambiguity.

**Disposition:** do not add a first-class core `lane_registration` record now.
Future lanes should emit a compact lane-local registration summary and reuse
existing common records. Reconsider promotion only if another lane cannot be
audited without duplicating a materially equivalent custom representation, or
if R4 cannot compare registration status across lanes.

### 3.4 Control outcome closure — sufficient evidence, real inspection friction

The scientific and integrity controls resolved in retained evidence:

- the registration freeze recorded applicability, resolution stage, exact
  evidence bindings, and pending execution legs for all common and lane
  controls;
- run records retained applicable control references and causal facts;
- the C02 audit closed twelve structural obligations and retained the
  medium-dependency, trace-shuffle, producer-parity, support/budget, baseline,
  isolation, and reconstruction audits; and
- the terminal record referenced all nineteen common and five L01 control IDs.

No missing control was silently counted as passed. However, the terminal record
contains control IDs rather than a single machine-resolvable disposition for
each ID. A reviewer must traverse the registration freeze, run records, cycle
audit, policy mapping, and terminal record to recover complete closure. This is
the one concrete R2 concern that became representational friction.

The first lane still does not justify one file per control or a new core schema
designed from a single realization. That would turn one observed traversal
problem into twenty-four new record instances and could overfit Option A.

**Disposition:** do not add first-class core `control_outcome` records now.
Starting with the next lane, require one compact lane-local control-resolution
index before terminal closure. It must map every mandatory control ID to:

```text
applicability
resolution stage
outcome status
evidence references
fail-closed effect
```

The index is a projection over retained evidence, not new evidence. It may be
embedded in or indexed by the lane closeout bundle. R4 will decide whether
recurrence across completed lanes justifies a common first-class schema. P2-I1
does not require a retrospective record because its result is already closed,
its evidence is recoverable, and R3 now records the concrete friction. If R4
needs a uniform cross-lane table, it may generate a non-evidential P2-I1
projection from the retained bundle without reopening or reclassifying C02.

### 3.5 Review and ceremony proportionality — adequate with a future constraint

P2-I1 correctly separated semantic decisions from deterministic validation,
but repeated gate identity work and duplicated explanations made the lane
harder to navigate. This did not corrupt the result. It did demonstrate that
later lanes should reuse the contract shape without reproducing every P2-I1
document and validator mechanically.

Future lane practice is therefore:

- keep one detailed evidence-expandable lane checklist;
- keep one cumulative decision record per lane rather than one file per
  decision;
- generate compact identity and control indexes from retained evidence;
- review new semantic assumptions once;
- close post-review fixes with targeted validation rather than recursive
  review; and
- add ceremony only when evidence use or a concrete failure requires it.

### 3.6 Lane ownership and anti-subsumption — passed

L01 remained the general history-conditioned later-possibility lane:

- multi-source co-conditioning remained blocked for L02;
- trace shuffle tested source-history specificity without claiming L03 trail
  persistence;
- reduced support did not become L04 nursery/support evidence;
- no boundary-exchange claim was assigned to L05;
- doubled carrier load did not become L06 circulation evidence; and
- the support contrast explicitly denied a parent-basin claim under L07.

The dormant alternatives record strengthens this boundary by assigning likely
topological/basin ownership to L05/L07 and pulse-contact-only ownership to L03
before any future preregistration.
