# AE01 P1-I5 Artifact, Tooling, and Reconstruction Contract

**Status:** revised and frozen at P1-I5 revision 0.24

**Tooling version:** `1.1.0`

**Dependency profile:** Python `>=3.11`, `jsonschema==4.26.0`

**Evidence effect:** infrastructure only; no lane execution or positive result

## 1. Owned implementation

P1-I5 introduces only experiment-local infrastructure:

- `scripts/ae01_tooling.py` contains canonicalization, digest, validation,
  manifest, runtime-binding, report, projection, and read-only guards.
- `scripts/ae01.py` is the single command entry point.
- `configs/p1_i5_profiles.json` contains versioned environment, command,
  dependency, and resource profiles.
- `configs/p1_i5_execution_policy.json` materializes the finite P1-I4
  comparison and stopping policy.
- `configs/p1_i4_developmental_interpretation_policy.json` freezes hard-gate
  partitioning, threshold relations, lane boundary ladders, classification
  value, and allowed next moves.
- `contracts/metric-sheets/` freezes the primary metric formula and candidate-
  blind resolution procedure for every lane.
- `contracts/fixtures/` separates schema-valid, semantic-negative, and invalid
  conformance cases.
- `implementation/tests/` verifies Phase 1 infrastructure.

No distribution metadata, installable package, reusable ecology source,
domain package, or PyGRC implementation is introduced. O-002 remains deferred.

## 2. Authority split

The tooling preserves the P1-I3 authority order:

```text
Markdown meaning             normative interpretation and claim boundary
JSON Schema Draft 2020-12    persisted envelope and field shape
Python semantic validation   cross-field and cross-record fail-closed checks
```

Python rejects invalid combinations but does not add a stronger claim or
change a field meaning. Revision 0.24 explicitly reopens P1-I3 and advances
the schema to `1.1.0` for first-class metric-sheet, metric-calibration, and
developmental-interpretation records. No retained `1.0.0` record migrates
silently.

## 3. Canonical serialization and digests

Canonical values mirror the PyGRC Python profile:

- mappings require string keys and sort lexicographically;
- lists preserve order;
- sets are ordered by canonical JSON representation;
- unsupported objects and non-finite floats fail;
- compact semantic JSON uses sorted keys, ASCII escaping, and separators
  `,` and `:` with no insignificant whitespace;
- tracked JSON uses two-space indentation, sorted keys, ASCII escaping, and one
  final newline; and
- SHA-256 is the only digest algorithm.

`output_digest` is SHA-256 over compact canonical JSON. `file_sha256` is
SHA-256 over exact file bytes. Non-JSON semantic artifacts use the exact-file
digest. IDs derived by tooling use:

```text
ae01:<normalized-kind>:<first-16-hex-of-canonical-identity-digest>
```

Accepted externally assigned stable IDs such as `AE01-L01` are never renamed
or regenerated by this policy.

## 4. Portable paths

Persisted paths are repository-relative POSIX strings or `.` for repository
root. Empty segments, traversal, home expansion, drive prefixes, backslashes,
and filesystem-root paths fail. Live machine locations may be command-line or
environment inputs but are never written into records, receipts, manifests, or
reports.

## 5. Evidence-use and retention policy

```text
exploratory_scratch -> registered_probe -> retained_evidence
```

- Scratch may inform design but cannot reference a claim or gate.
- A registered probe has frozen inputs and controls but still cannot reference
  a claim or gate.
- Retained evidence requires `last_verified_status=passed`, a fully resolved
  manifest, expected semantic/file digests, size, profiles, reconstruction, and
  claim dependencies.

Tracked selected evidence belongs under the owning experiment. Disposable
runs, temporary receipts, duplicate reconstructions, and validation summaries
belong under ignored top-level `outputs/`. An individual artifact above 1 MiB
or selected set above 10 MiB requires explicit retention review. A local-only
large artifact remains declared as `reconstructable_local_only` with the same
verified reconstruction contract.

## 6. Shared profiles and resolved manifests

Authored manifests reference versioned environment, command, dependency,
resource, and, for live work, realization profiles. Before retention, manifest
generation embeds every referenced shared profile and applicable realization
profile. Semantic validation checks profile types, source revisions, artifact
sizes, digests, tiers, claim dependencies, and live realization applicability.

Two reconstructions from the same declared inputs must produce identical
canonical records and `output_digest`. Deterministic JSON files should also
produce the same `file_sha256`. A mismatch reopens every dependent claim.

## 7. Finite comparison policy

Every lane has exactly seven frozen comparison cells and five lane-specific
controls. The five-control symmetry is local to AE01. All seven P1-I4 logical
comparison groups are explicitly covered: Lane 2's access/capacity contrast
serves both transfer and budget/leakage purposes, while Lane 7 records
budget/leakage as inapplicable because that lane classifies demand without an
economy claim. `AE01-CTRL-09` remains required in Lane 7, and an economy-bearing
realization must reopen the exception. All nineteen common controls remain
required; P1-I4 “especially direct” controls receive `direct` priority while
the rest receive `inherited` priority in the fully resolved policy.

Resolution gives every cell a deterministic configuration ID and embeds its
seed/no-seed policy, attempt and infrastructure-retry limits, resource
envelope, required artifact roles, success criteria, invalid-run criteria, and
infrastructure-failure criteria. An infrastructure failure remains visible and
can never satisfy a scientific signature.

Candidate-blind metric calibration uses seeds `19`, `43`, `71`, `109`, and
`163`; candidate cells remain blocked until their resolution `delta` is frozen.
Live candidate cells use deterministic seeds `101`, `211`, and `307`, one attempt per
seed, and at most one infrastructure-only retry. A cell has 120 seconds,
512 MiB memory, and 256 MiB disk. The highest support rung retains these
anchors:

```text
100% mandatory signatures
100% mandatory controls
100% directional seed agreement
strict normalized candidate/control margin > 0
at least 2 of 3 transfer-contrast seeds
zero unsafe flags
```

These anchors do not form an accept/reject filter. Each result also records its
machine-derived `robust_aligned`, `narrow_aligned`, `resolution_limited`,
`mixed_direction`, `narrow_counter`, `robust_counter`, `resolution_unknown`,
or `not_applicable` relation; its lane boundary rung; support status;
classification value; and two-axis developmental interpretation. The strict
positive threshold establishes direction only. P1-I4 lane ceilings still
control interpretation. Threshold, calibration, seed, cell, resource, or
retry changes after candidate observation require contract revision and
affected rerun.

A valid scientific result may propose a bounded local refinement or alternative
realization. That proposal requires a new preregistration and may never consume
the one infrastructure-only retry or rewrite the prior result.

Synthesis fails until exactly one terminal record exists for each stable lane.
Each terminal record must reference a same-lane developmental interpretation.
The eight explicit non-selection conditions remain fixed; non-selection does
not erase atlas observations or next questions.

## 8. Runtime binding and realization

Artifact inspection imports no PyGRC and is explicitly non-runtime. Every live
request first requires a realization profile that explicitly marks the
realization available and enabled. It then:

1. names `pygrc_runtime_with_rcae_producer` or `pygrc_native_runtime`;
2. references a realization profile;
3. confirms the execution class and requested operations are allowed by that
   unchanged profile;
4. imports the locally installed/configured `pygrc` runtime;
5. compares exact distribution identity and required public API surfaces;
6. emits a receipt whether binding passes or fails;
7. records no machine-local path;
8. performs no fallback to inspection, mocks, or another realization; and
9. optionally wraps the local graph checkout in a content fingerprint guard
   that raises on any worktree-file change.

A failed binding uses observed identity `pygrc:unavailable` or the actual
incompatible identity, records `conformance_status=failed`, performs no state
operation, and remains blocked/incomplete rather than negative scientific
evidence.

PyGRC location is configured locally through the active Python environment or
`PYTHONPATH`. It is not committed. A native surface becoming available never
changes an RCAE-constructed profile; native transition requires a new profile,
explicit transition record, and rerun.

## 9. Report authority

Generated facts come only from controlling machine records.
`generated_projection` rejects authored interpretation.
`assembled_interpretation` deterministically combines generated facts with a
separate authored Markdown input and rejects declared blocked statements.
Reports retain their claim-boundary reference and cannot alter terminal,
control, debt, score, or selection facts.

## 10. Lane projections

Validation requires exactly seven unique registry rows in stable order, orders
1 through 7, current names, and `validated` status for every target. Missing,
duplicate, stale, or reordered projections fail. The plan, roadmap, and AE01
README are checked before P1-GATE.

## 11. Reconstruction commands

Run from repository root. The pinned dependency is acquired ephemerally; it
does not create distribution metadata.

Validate all Phase 1 records, projections, policy, and fixtures:

```bash
uv run --with jsonschema==4.26.0 python experiments/2026-07-AE01-post-n30-demand-composition-atlas/scripts/ae01.py validate-phase1
```

Run focused infrastructure tests:

```bash
uv run --with jsonschema==4.26.0 python -m unittest discover -s experiments/2026-07-AE01-post-n30-demand-composition-atlas/implementation/tests -v
```

Materialize the resolved finite policy:

```bash
uv run --with jsonschema==4.26.0 python experiments/2026-07-AE01-post-n30-demand-composition-atlas/scripts/ae01.py resolve-policy --output outputs/ae01-p1-i5-resolved-policy.json
```

Freeze a candidate-blind resolution calibration and produce its frozen metric
sheet:

```bash
uv run --with jsonschema==4.26.0 python experiments/2026-07-AE01-post-n30-demand-composition-atlas/scripts/ae01.py freeze-resolution --metric-sheet experiments/2026-07-AE01-post-n30-demand-composition-atlas/contracts/metric-sheets/AE01-L01.json --calibration-input experiments/2026-07-AE01-post-n30-demand-composition-atlas/contracts/fixtures/inputs/l01-candidate-blind-calibration-input.json --calibration-output outputs/ae01-l01-metric-calibration.json --frozen-sheet-output outputs/ae01-l01-frozen-metric-sheet.json
```

Classify per-seed candidate margins without collapsing them into a scalar gate:

```bash
uv run --with jsonschema==4.26.0 python experiments/2026-07-AE01-post-n30-demand-composition-atlas/scripts/ae01.py classify-margin --metric-sheet outputs/ae01-l01-frozen-metric-sheet.json --margin 101:0.08 --margin 211:0.03 --margin 307:0.06
```

Produce and compare duplicate validation summaries:

```bash
uv run --with jsonschema==4.26.0 python experiments/2026-07-AE01-post-n30-demand-composition-atlas/scripts/ae01.py validate-phase1 --output outputs/ae01-p1-i5-validation-a.json
uv run --with jsonschema==4.26.0 python experiments/2026-07-AE01-post-n30-demand-composition-atlas/scripts/ae01.py validate-phase1 --output outputs/ae01-p1-i5-validation-b.json
uv run --with jsonschema==4.26.0 python experiments/2026-07-AE01-post-n30-demand-composition-atlas/scripts/ae01.py digest --input outputs/ae01-p1-i5-validation-a.json
uv run --with jsonschema==4.26.0 python experiments/2026-07-AE01-post-n30-demand-composition-atlas/scripts/ae01.py digest --input outputs/ae01-p1-i5-validation-b.json
```

Live runtime, manifest, and report commands require their lane-local profile,
descriptor, and projection records and are documented in `../scripts/README.md`.
They fail rather than inventing missing inputs.

## 12. P1-I5 claim boundary

Successful validation proves that the infrastructure enforces the frozen
contracts on its fixtures. P1-GATE may authorize candidate-blind calibration
and lane registration, but candidate execution remains lane-blocked until its
metric sheet is frozen. Validation does not establish a shared-medium pattern,
successful composition, reusable primitive, PyGRC compatibility on another
machine, catalog admission, or N31+ selection. `AE01-C0` remains the highest
assigned rung until P1-GATE review separately assigns C1/C2.
