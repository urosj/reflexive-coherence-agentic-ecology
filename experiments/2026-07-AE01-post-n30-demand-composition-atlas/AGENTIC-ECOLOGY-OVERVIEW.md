# Agentic Ecology: From Theory to Controlled Experiments

**Audience:** readers who want to understand how this repository moves from
agentic-ecology concepts to bounded LGRC-backed evidence

**Role:** general narrative orientation; controlling scientific claims remain
in experiment contracts, retained evidence, closeouts, and owner decisions

**Current program:** [AE01 agentic-ecology view](AE01-AGENTIC-ECOLOGY-OVERVIEW.md)

## The short version

Most agent systems begin with already-formed agents. Each agent has internal
state, receives information, decides, acts, and communicates. The environment
is usually the stage on which those agents operate.

Agentic ecology begins one level earlier:

> What shared geometry makes agent-like distinctions, memory, relation,
> selective response, coordination, and continued identity possible at all?

In this view, an agent is not first a sealed object with a state machine. It is
a local differentiation that persists within a larger field of support,
boundaries, costs, traces, routes, susceptibilities, reserve, and history. The
environment is not passive background. It carries conditions that shape what
can form, continue, interact, or fail next.

This changes the engineering question. Instead of asking only:

```text
What rule should agent A use to send an instruction to agent B?
```

we also ask:

```text
What shared medium can A alter?
How does that alteration persist and remain attributable?
Why does it change B's later possibilities?
Why does B respond from its own geometry rather than obeying a command?
What does maintaining that relation cost?
```

The experiments in this repository turn those questions into frozen
comparisons, causal controls, reproducible runtime evidence, bounded
interpretations, and explicit implementation debt.

## What “agentic” and “ecology” mean here

“Agentic” does not mean that the project begins by assigning choice,
intention, goals, or consciousness to an object. It points toward the capacity
of a local or higher-order identity to continue through perturbation by using
and reshaping available support, boundaries, coupling, memory-like
aftereffects, and reserve.

“Ecology” does not mean merely placing many agents in one simulation. It means
that local continuations become mutually conditioned through a shared economy
of geometry, support, cost, trace, capacity, and history. Participants and
environment cannot be fully explained as independent objects plus messages
between them.

An agentic ecology is therefore not “a multi-agent system with richer state.”
It is a system in which participant-like identities and their relations are
carried increasingly by the same evolving substrate. Controlled experiments
ask which parts of that statement can be made operational and which remain
theory, scaffolding, or missing substrate demand.

## The theory-to-evidence bridge

The conceptual papers define a target ontology. They translate familiar agent
language into coherence geometry:

| Familiar language | Agentic-ecology reading |
| --- | --- |
| agent state | basin or boundary condition, support, reserve, susceptibility |
| action | costly transformation of geometry, coupling, support, or flux |
| memory | persistent geometry, event history, or aftereffect |
| communication | participation through a shared medium |
| coordination | coupled local responses through shared conditions |
| environment | active nested ecology rather than passive background |

That ontology is useful, but vocabulary is not evidence. Renaming a message
bus as a field, a global variable as a parent basin, or a state machine as
coherence geometry does not create an agentic ecology.

The graph project supplies another layer: PyGRC implementations and controlled
evidence about graph-native transport, event histories, local differentiation,
route selection, traces, eligibility, restoration, and bounded shared-medium
participation. But graph evidence does not automatically establish an
ecology-scale function. A trace is not yet a niche; common storage is not yet
a functional shared pool; a packet route is not yet circulation.

The ecology experiments sit between them:

```text
agentic-ecology theory
  -> names a functional ecological relation

graph/PyGRC evidence
  -> supplies admitted mechanisms, limits, and debts

ecology experiment
  -> constructs and discriminates the relation

classified result
  -> records what appeared, what remained scaffolded, and what is missing

future graph experiment
  -> may test a recurring substrate demand natively
```

This is a spiral rather than a one-way dependency. Ecology experiments consume
what PyGRC can already express, but they may also use a declared producer or
adapter when the required relation is missing. That construction can expose a
precise graph-side need. A later native implementation does not retroactively
upgrade earlier scaffolded evidence.

## How to read the claims

The project keeps several statement types separate because moving between
them requires new evidence:

| Layer | Meaning |
| --- | --- |
| Conceptual motivation | A paper identifies a meaningful relation and vocabulary |
| Inherited graph evidence | Existing PyGRC work establishes bounded substrate capabilities and limits |
| Constructed ecology mechanism | RCAE assembles an explicit probe from admitted surfaces plus declared scaffolding |
| Observed experiment result | Frozen execution and controls establish what happened in that realization |
| Ecology interpretation | The observed relation receives bounded ecological meaning |
| Admitted reusable/native mechanism | A substrate abstraction has independent native evidence and an explicit admission decision |

These layers are not automatic maturity stages. A useful ecology
interpretation does not make its constructed mechanism native. A native
packet or feedback operation does not make the entire ecology relation native.
A request for a missing surface is not evidence that the surface exists.

## What a controlled agentic-ecology experiment must expose

A useful experiment must identify more than a successful output. It should
make the following relationships inspectable:

1. **Physical opportunity:** which participants, routes, costs, boundaries,
   and shared surfaces exist before the candidate relation.
2. **Carrier:** where the proposed ecological condition actually lives—current
   state, ordered history, trace, support, boundary condition, reserve, or a
   declared composite.
3. **Later consequence:** what formation, continuation, access, response,
   susceptibility, or failure changes afterward.
4. **Causal intervention:** what happens when the carrier is withdrawn,
   clamped, shuffled, diverted, privatized, or replaced.
5. **Alternative explanations:** whether direct addressing, a global
   variable, arbitrary labels, a controller, quantity alone, or fixture logic
   can reproduce the output.
6. **Provenance:** which native events, producer actions, and adapter-owned
   state carried each causal leg.
7. **Restoration and reconstruction:** whether save/load/reset and retained-
   only analysis preserve the declared identity and result.
8. **Claim ceiling:** what the evidence supports and which attractive names
   remain prohibited.

This is why the experiment workspaces contain more than scripts and summary
tables. The detailed contracts and receipts separate similar outputs produced
by different causal architectures.

## Native machinery, producers, and debt

The project follows a native-first rule:

```text
try the relation with admitted native machinery
  -> if adequate, retain the native causal path
  -> if missing or inadequate, use the smallest explicit producer/adapter
  -> record exactly what that scaffold owns
  -> extract the missing native requirement only after evidence
```

Producer assistance is not automatically failure. It is useful when it makes
an otherwise missing causal relation executable without hiding the boundary.
The failure would be to call scaffold-owned behavior native, let the producer
author the scientific answer, or erase the debt after a positive result.

Repeated missing machinery across independently controlled experiments may
justify a new graph-side implementation proposal. One attractive fixture does
not.

## What the evidence can contribute

Controlled agentic-ecology work can contribute at least four distinct things:

- a bounded ecological relation that survives registered causal controls;
- a negative or mode-specific result that reveals which geometry does not
  carry the function;
- a reusable experimental discriminator for distinguishing lookalike
  mechanisms; and
- a precise naturalization or medium debt that later work can compare across
  experiments.

The work does not need to prove a complete ecology to be valuable. It can show
that shared history changes participant-relative possibility, that several
events constitute one later-relevant carrier, that present state and formation
history compute different functions, or that an apparent operation-composition
effect is really a simpler temporal rule.

## Current experimental program: AE01

AE01 is the Post-N30 Demand and Composition Atlas. It separates seven
ecology-scale questions—niche conditioning, shared-pool co-conditioning,
trace dynamics, support, boundary-conditioned exchange, capacity circulation,
and parent-basin modulation—so that success in one cannot silently answer the
others.

Its first two lanes now provide controlled results:

- P2-I1 establishes a bounded shared-medium differential niche-conditioning
  demand pattern.
- P2-I2 establishes bounded shared-pool co-conditioning through state,
  scaffolded active history, and hybrid carriers, followed by appendices that
  distinguish live operation complementarity from a terminal-token temporal
  gate.

AE01 as a whole remains at `AE01-C2`; the remaining five lanes and cross-lane
recurrence/non-selection stages have not been completed. Read the
[AE01-specific agentic-ecology overview](AE01-AGENTIC-ECOLOGY-OVERVIEW.md) for
the current geometry, results, limits, and next questions. For the complete
P2-I2 case-by-case interpretation, use the
[P2-I2 interpretative synthesis](reports/P2-I2-interpretative-synthesis.md).
