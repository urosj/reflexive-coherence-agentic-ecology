# Contributing

This repository is for Reflexive Coherence agentic-ecology theory, translation
method, vocabulary, and claim-boundary documentation.

Appropriate contributions include:

- fixes or clarifications to papers in `papers/`;
- improvements to source grounding, citations, and cross-repository links;
- clearer examples of state-rule to RC-geometry translation;
- tighter producer-residue and naturalization-debt language;
- documentation fixes, broken-link fixes, and license/citation corrections.

Implementation work for graph runtimes, telemetry, visualization, or LGRC model
families should normally go to
[graph-reflexive-coherence](https://github.com/urosj/graph-reflexive-coherence).
Core RC/FRC theory and Arc of Becoming changes should normally go to
[geometric-reflexive-coherence](https://github.com/urosj/geometric-reflexive-coherence).

## Claim discipline

Keep conceptual, scaffolded, and implemented claims separate.

When adding or revising text, state whether the claim is:

- conceptual vocabulary;
- engineering method;
- producer-mediated design;
- partially naturalized substrate behavior;
- implemented and artifact-backed evidence.

Do not upgrade a conceptual mapping into an implementation claim without linking
to the implementation repository, runnable path, report, or artifact that
supports it.

## Workflow

1. Create a focused branch.
2. Keep changes scoped to one paper, section, or metadata surface when possible.
3. Use relative links for local repository files.
4. Avoid home-directory paths, drive-letter paths, IDE-specific links, and
   machine-local artifact references.
5. Run a Markdown link or prose review when practical.
6. Open a pull request with a concise summary, claim-boundary impact, and any
   verification performed.

## Paper style

- Prefer direct definitions over metaphor when the text is making a claim.
- Preserve the distinction between RC target ontology and producer scaffolding.
- Keep examples explicit about what is native, scaffolded, or aspirational.
- Use tables when they clarify a translation ledger or claim ladder.

## Conduct

Please follow [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md). Keep discussion focused
on making the RC agentic-ecology material clearer, better grounded, and easier
to inspect.
